---

### 开发顺序建议
1. **基础设施 (Module 1)**：必须优先完成，否则后续无法进行。
2. **数据采集 (Module 2) 与 语义处理 (Module 3)**：可并行开发，先有数据才能建图。
3. **混合检索 (Module 4)**：在图谱和向量库有基础数据后开始。
4. **Agent 逻辑 (Module 5)**：作为核心大脑，依赖于下方所有模块提供的检索能力。
5. **UI (Module 6)**：可与 Module 5 并行，用于展示最终成果。

### 验证方法说明
- **自动化测试**：每个模块需包含对应的 `tests/` 目录，通过 `pytest` 运行。
- **人工验证 (👤)**：涉及 API 配置、Schema 设计及主观体验（推送质量）的部分，必须由开发者手动确认并签字。
