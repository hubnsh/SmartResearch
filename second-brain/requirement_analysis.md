# 个人知识管理 Agent (Second Brain) 需求分析文档

## 1. 项目概述
本项目旨在构建一个智能化的个人“第二大脑”（Second Brain）。不同于传统的被动式笔记软件，本项目通过 Agent 技术实现知识的自动整理、深度关联与主动推送。它能够从多种渠道（Obsidian、Notion、网页、书摘等）捕获信息，利用图数据库（Neo4j）构建复杂的知识网络，并通过 RAG（检索增强生成）技术提供精准的知识问答与关联建议。

## 2. 核心功能需求

### 2.1 多源数据自动化采集
- **Obsidian/Notion 同步**：通过 API 实时或定时同步用户笔记内容。
- **网页与书摘捕获**：集成浏览器插件或微信剪藏，自动提取网页核心内容、书摘元数据。
- **自动格式化**：将非结构化数据转化为统一的 Markdown/JSON 格式，便于后续处理。

### 2.2 知识图谱构建 (Knowledge Graph)
- **实体提取**：利用 LLM 从笔记中提取人名、地名、概念、项目等实体。
- **关系建模**：识别笔记间的引用关系、标签共现关系及语义相关性。
- **Neo4j 存储**：将实体与关系存储于 Neo4j 数据库，实现知识的可视化与多级路径检索。

### 2.3 智能化 RAG (检索增强生成)
- **向量化存储**：将笔记片段进行 Embedding 向量化，存储于向量数据库。
- **混合检索**：结合关键词搜索、向量相似度搜索与图路径搜索，提高检索精度。
- **语境问答**：基于全量个人知识库回答用户问题，并引用来源。

### 2.4 主动式 Agent 能力 (核心亮点)
- **自主规划**：Agent 能够根据用户当前的关注点（如正在写的文档），自主决定检索哪些历史笔记。
- **关联推送**：在用户浏览或输入时，主动推送“你可能感兴趣的相关知识”或“过去相似的思考”。
- **知识整理建议**：定期扫描知识库，发现冲突、重复或可以合并的知识点，并向用户提出优化建议。
- **定时任务**：通过定时任务执行知识库的索引更新、图谱优化及每日知识回顾简报。

## 3. 技术栈选型
- **数据源层**：Obsidian API, Notion SDK, Web Scraper (Playwright/Puppeteer)
- **逻辑编排层**：LangChain / LangGraph (用于实现 Agent 的自主规划与状态管理)
- **数据库层**：
  - **关系/图数据库**：Neo4j
  - **向量数据库**：ChromaDB / Milvus / Pinecone
- **大模型**：GPT-4o / Claude 3.5 / DeepSeek-V3 (用于推理、提取与生成)
- **基础设施**：Python (FastAPI), Celery / APScheduler (定时任务)

## 4. 业务流程
1. **输入阶段**：用户新增笔记或收藏网页。
2. **处理阶段**：
   - 触发 Webhook 或定时任务。
   - LLM 提取摘要、标签及关联实体。
   - 更新向量数据库与 Neo4j 图谱。
3. **交互阶段**：
   - 用户提问 -> RAG 检索 -> 生成回答。
   - 用户创作 -> Agent 监控上下文 -> 主动推送关联内容。
4. **维护阶段**：定时任务清理过期信息，优化图谱拓扑结构。

## 5. 成功指标 (KPI)
- **自动化程度**：知识录入到图谱生成的自动化率 > 90%。
- **检索准确率**：RAG 回答的准确性及引用相关性。
- **推送价值感**：用户对主动推送内容的采纳率或互动率。
- **系统性能**：单次检索响应时间 < 2s。

## 6. 后续演进
- 支持语音输入与多模态知识处理（图片、视频笔记）。
- 离线化部署支持（Llama-3 本地化运行以保护隐私）。
- 跨设备同步与移动端交互。
